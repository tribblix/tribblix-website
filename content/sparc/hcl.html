--- title: SPARC hardware compatibility ---

> illumos on SPARC

The following is a record of SPARC systems that are known to work, or known to not work, with illumos. Be aware that there may be subtle differences in configuration or firmware that may lead to alternate results.

Working graphics cards are few and far between. There have been no recent reports of successfully using the local graphics cards to run an X session.

A reasonable memory configuration is a minimum of 1G. While it's possible to install Tribblix in 768M, that's the lower limit with the current builds.

Reports regarding success or otherwise on other models are welcome, please let us know on the mailing list

Known to work

  • T4-1, T4-2
  • T5240/T5140
  • T5120/T5220
  • T1000/T2000
  • SunBlade 2000
  • SunBlade 1500
  • SunBlade 150
  • SunFire V240
  • SunFire V120
  • Sun Netra X1
  • E450
  • Ultra 5
  • Ultra 2
  • Ultra 1 (not the 140MHz one, though)

Note that using the IDE controllers on some of the newer entry-level desktops can be problematic.

On the sun4v systems, installation on either bare metal or inside an LDOM is supported. There is as yet no support for running as a control domain (ie, for hosting other LDOMs).

Should work, but don't

The following systems should be supported, but do not currently work. There's a suspicion that gcc builds here are faulty, and old Studio builds work.

  • M-series (Olympus, M3000/M4000/M5000)
  • T5440 (if fully populated)
  • T4-4 (if fully populated)
  • T3-2 (if fully populated)
  • Ultra 25/45

Will not work

The following systems will not work, as there is no support in the codebase for them.

  • 32-bit systems (sun4m, sun4d, sun4c,...)
  • T5, T7, M7/M8
  • Enterprise servers (E3000/E3500-E6000/E6500)
  • StarFire/StarCat (E10K, F15K, F25K)
  • Telco (CT) systems
